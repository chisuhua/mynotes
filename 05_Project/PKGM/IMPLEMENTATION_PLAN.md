## 1. 执行摘要

本实施计划基于已确认的 6 个 ADR 和架构设计文档，分三个阶段完成 PKGM 系统建设：

| 阶段 | 周期 | 目标 | 关键交付物 |
|------|------|------|-----------|
| **Phase 1** | 1-2 周 | 基础建设 | 目录骨架 + Schema + 模板 |
| **Phase 2** | 2-3 周 | Agent 管线 | 摄取/抽取/生成自动化 |
| **Phase 3** | 2-3 周 | 检索与展示 | 图谱查询 + 健康扫描 |

**总体周期**: 5-8 周  
**关键里程碑**: Phase 1 完成后即可手动使用，Phase 2 实现自动化，Phase 3 完善体验

## 4. 风险与缓解措施

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| **Schema 设计缺陷** | 后期需要重构 | 中 | Phase 1 创建测试页面验证，小步迭代 |
| **关系抽取准确率低** | Wiki 质量不可靠 | 高 | 人工确认环节，置信度标记，持续优化 Prompt |
| **目录结构膨胀** | 难以维护 | 中 | 定期重组，Agent 建议合并 |
| **Agent 执行时间长** | 用户体验差 | 中 | 增量处理，缓存机制，异步通知 |
| **Graphify 集成复杂** | Phase 2 延期 | 低 | 作为可选功能，不影响核心管线 |
| **ChromaDB 性能问题** | 语义搜索慢 | 低 | Phase 3.2 再评估，可换 FAISS |
| **OpenClaw 能力限制** | 部分功能无法实现 | 低 | 提前调研，必要时自定义技能 |
