## 1. 背景（修订）

原始设计将 04_Knowledge 定位为"原创知识"，但实际工作流需要：
1. **两阶段消化**：从原材料到结构化 Wiki 需要渐进式处理
2. **动态目录**：知识领域会随研究深入不断扩展
3. **原创分离**：Suhua 的原创思考需要独立空间，不与提取内容混用

---

### 2.2 知识流向

```
┌─────────────────┐
│ 00_Raw_Sources  │  外部素材（论文/文章/代码）
│   (原材料)      │
└────────┬────────┘
         │ Ingest Stage 1
         ▼
┌─────────────────┐
│ 04_Knowledge    │  LLM 初步消化
│   (初步消化)    │  - 按研究领域组织（12 个一级目录）
│                 │  - 子目录动态扩展
│                 │  - 简化 frontmatter
│                 │  - 状态标记：draft → reviewed → archived
└────────┬────────┘
         │ Ingest Stage 2
         ▼
┌─────────────────┐
│ 01_Wiki         │  深度咀嚼
│   (结构化 Wiki)  │  - 完整 Schema 合规
│                 │  - 类型化关系
│                 │  - 链接回 04_Knowledge（溯源）
└─────────────────┘

┌─────────────────┐
│ 06_Mynotes      │  Suhua 原创思考
│   (原创内容)    │  - 架构设计
│                 │  - 设计决策
│                 │  - 实验记录
│                 │  - 不经过 Ingest 流程
└─────────────────┘
```

### 2.3 04_Knowledge 动态目录规则

**一级目录**（预设，12 个）：
- 按知识领域划分，对应 USER.md 中的技术栈

**二级/三级目录**（动态扩展）：
- LLM 在提取时发现新子领域，自动创建对应子目录
- Suhua 可手动重组/合并目录
- Agent 定期建议目录优化方案

**示例**：
```
04_Knowledge/
└── 01_GPU_Architecture/
    ├── generations/         # 预设
    ├── microarchitecture/   # 预设
    ├── simt-execution/      # 动态创建（发现 SIMT 相关论文）
    └── memory-hierarchy/    # 动态创建（发现缓存相关论文）
```

### 2.4 页面状态流转（04_Knowledge）

| 状态 | 含义 | 触发条件 |
|------|------|---------|
| `draft` | 刚提取，待审核 | LLM 完成 Stage 1 提取 |
| `reviewed` | 已审核，待提炼 | Suhua 确认内容准确 |
| `archived` | 已提炼到 Wiki | Stage 2 完成，页面保留为历史记录 |

```yaml
# 04_Knowledge 页面 frontmatter 示例
---
title: "[提取] CUDA Warp Divergence"
type: concept
domain: "01_GPU_Architecture"
source_type: primary
source_ref: "NVIDIA CUDA C Programming Guide"
confidence: 3
status: draft  # draft | reviewed | archived
created: 2026-04-14
created_by: "agent/ingest"
---
```

### 2.5 01_Wiki 链接回 04_Knowledge

Wiki 页面必须标注提取来源：

```yaml
# 01_Wiki 页面 frontmatter 示例
---
title: "CUDA Warp Divergence"
type: concept
domain: "01_GPU_Architecture"
source_type: primary
source_ref: "NVIDIA CUDA C Programming Guide"
confidence: 4
extracted_from: "[[04_Knowledge/01_GPU_Architecture/simt-execution/warp-divergence]]"
relations:
  depends_on:
    - "[[SIMT Execution Model]]"
---
```

### 2.6 06_Mynotes 原创内容规范

**定位**：Suhua 的原创思考，不经过 Ingest 流程

**目录结构**：
```
06_Mynotes/
├── architecture/       # 架构设计草稿
├── decisions/          # 设计决策记录
├── experiments/        # 实验验证报告
├── reflections/        # 思考笔记
└── todo/               # 待整理内容
```

**与 04_Knowledge 的区别**：
| 维度 | 04_Knowledge | 06_Mynotes |
|------|-------------|-----------|
| 来源 | 外部素材提取 | Suhua 原创 |
| 格式 | 简化 frontmatter | 自由格式 |
| 状态 | draft → reviewed → archived | 无状态要求 |
| 目标 | 提炼到 01_Wiki | 永久保存 |

## 3. 影响分析

### 正面影响

| 影响 | 说明 |
|------|------|
| 渐进式处理 | 不追求一步到位，允许"先提取再整理" |
| 动态目录 | 知识结构自然生长，不预设过多子目录 |
| 溯源清晰 | 01_Wiki 链接回 04_Knowledge，可追溯提取过程 |
| 原创分离 | 06_Mynotes 独立，不与提取内容混用 |

### 风险

| 风险 | 缓解措施 |
|------|---------|
| 04_Knowledge 目录膨胀 | 定期重组，Agent 建议合并相似主题 |
| 两阶段边界模糊 | 明确 Stage 1/Stage 2 输出规范 |
| 06_Mynotes 被忽略 | 在 01_Wiki/index.md 中添加"原创思考"入口 |
