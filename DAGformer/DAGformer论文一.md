一、 论文核心摘要

本论文提出DAGformer，一种为有向无环图（DAG）数据结构量身定制的、全新的Transformer基础架构。与将DAG强行适配于现有序列或图模型不同，我们主张：DAG独特的方向性、无环性与层次性应被内化为模型的一阶归纳偏置。为此，我们设计了三大核心技术模块：1）拓扑约束的稀疏注意力，确保信息严格沿DAG的有向路径流动；2）层次-因果门控模块，动态调节跨层级与因果节点的信息交互强度；3）面向嵌套结构的动态路由机制，为处理节点本身是子DAG的复杂场景提供接口。我们在视觉场景图生成、文本抽象语义表示解析和智能体任务规划评估三个异构任务上进行了系统性实验。结果表明，DAGformer不仅在各项标准指标上显著超越现有序列与图神经网络基线，更展现出卓越的计算效率与结构可解释性。本工作为构建统一的结构化理解与推理模型迈出了关键一步，并为通向具备复杂规划能力的下一代智能体奠定了坚实的架构基础。

二、 关键技术思想脉络

思想演进遵循“解构问题 -> 确立原则 -> 构建模块”的逻辑链条：

1. 问题根源：标准Transformer的全连接注意力破坏了DAG的方向性与无环性，导致信息流混乱；通用图神经网络（GNN）的“消息传递”范式对DAG中至关重要的长程依赖和层次结构建模乏力。
2. 核心原则：处理DAG的架构必须尊重其拓扑排序所蕴含的因果/时序流。线性化输入顺序（如拓扑序、层次序）并非一种妥协，而是一种应被注意力机制显式利用的强结构性先验。
3. 解决方案：我们提出一个三位一体的架构范式：
   · 结构化的输入：根据任务语义选择最优DAG序列化策略（如层次序用于场景，拓扑序用于规划）。
   · 结构化的注意力：通过硬性掩码强制注意力矩阵与DAG的可达性矩阵对齐，实现“有向无环”的信息传播。
   · 结构化的门控：引入基于节点相对位置与关系类型的门控，让模型学会区分“包含”、“因果”等不同语义的边。

第3章：DAGformer方法论

· 3.1 预备知识与符号定义：正式定义DAG，并介绍三种核心的DAG序列化策略：拓扑序（用于因果流）、层次广度序（用于层次结构）、深度优先序（用于路径探索）。
· 3.2 拓扑约束的稀疏注意力
  · 核心思想：将DAG的可达性矩阵 M（若节点i可达j则M_ij=1）转化为注意力掩码。对于线性化序列中的位置i和j（i < j），仅当M_ij = 1时，才允许i关注j。这保证了信息只能从祖先节点流向子孙节点，且绝不自环。
  · 公式与实现：
    ```python
    # 伪代码示例
    # A: 原始注意力分数矩阵 [seq_len, seq_len]
    # M: 基于DAG构建的布尔可达性矩阵 [seq_len, seq_len]
    mask = (1 - M) * -1e9  # 将不可达位置掩码为负无穷
    constrained_attention = softmax(A + mask)
    ```
  · 优势：该设计在数学上严格保证了注意力机制对DAG方向性与无环性的遵从，计算复杂度从O(N^2)降低至接近O(E)（边数）。
· 3.3 层次-因果门控模块
  · 核心思想：并非所有有向边都具有相同语义。“人-手-手”是部分与整体的层次包含关系，而“开关-开灯”是因果触发关系。我们设计一个可学习的门控向量，根据节点对的相对深度差（Δdepth） 和边类型嵌入（e_type），来调制注意力权重的强度。
  · 公式：g_{ij} = σ(Linear([Δdepth_{ij}; e_type_{ij}]))，其中σ为Sigmoid函数。最终信息流为：Output_{j} += g_{ij} * AttentionHead(Q_i, K_j, V_j)。
  · 可视化：计划提供热力图，展示模型在处理“杯子在桌子上”和“人用刀切面包”时，对“on”和“using”两种关系赋予的不同门控强度。
· 3.4 动态路由与嵌套DAG处理（前瞻设计）
  · 核心思想：为处理节点本身是子DAG的复杂情况，我们引入一个轻量级路由网络。该网络为每个节点计算一个概率分布，决定其特征应更多地参与到“细粒度”（同层节点间）还是“粗粒度”（作为超节点与其它超节点间）的注意力计算中。
  · 实现：在本文中，该机制作为可扩展性设计进行阐述，并在消融实验中以简化版本（固定聚类）验证其潜力，为未来处理复杂规划图预留接口。
