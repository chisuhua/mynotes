🧪 P3 实验设置具体细节

一、实验目标
验证 Meta-GoT 在以下方面的能力：
1. 策略抽象能力：能否从具体推理图中提取通用策略图。
2. 策略迁移能力：能否将已有策略应用于新任务。
3. 自进化能力：随着任务经验积累，系统性能是否持续提升。
4. 跨领域泛化能力：在未见过的任务类型中是否具备快速适应能力。

二、实验任务与数据集
领域   任务   数据集   样本量   任务复杂度
数学   定理证明   MiniF2F + 自建几何证明集   1,200 题   高（多步推理）
法律   判例推理   CaseHold + 自建法律类比库   800 案例   中高（逻辑+语义）
科学   假设生成   SciFact + 自建科研问题集   600 问题   高（开放性）
军事   策略推演   MIST + 自建兵棋推演场景   400 场景   极高（对抗性）
伦理   两难决策   MoralScenarios + 自建   500 情境   中（价值权衡）
规划   多步任务规划   ALFWorld + 自建   700 任务   中高
代码   程序合成   HumanEval + MBPP   1,000 题   高（语法+逻辑）
医疗   诊断推理   MedQA-USMLE + 自建病例   600 病例   极高（风险敏感）

说明：
- 所有数据集均经过去重、标准化、难度标注。
- 每个任务设置 训练集（70%）、验证集（15%）、测试集（15%）。
- 跨领域迁移任务：在某一领域训练，另一领域测试（如：数学策略 → 法律推理）。

三、基线模型（Baselines）
模型   类型   说明
GoT-Engine (P2)   图式分治   使用固定策略库，无自进化
AutoGPT   自主代理   基于目标分解，无策略抽象
AlphaZero (变体)   强化学习   仅用于对抗类任务（如军事）
LLM + Chain-of-Thought   提示工程   GPT-4 + CoT
LLM + Tree-of-Thought   推理树   ToT 原始实现
Meta-RL (PPO + GNN)   元强化学习   作为策略优化基线

所有模型均在相同硬件与数据集上运行，确保公平对比。

四、评估指标（Evaluation Metrics）

1. 核心性能指标
指标   定义   计算方式
任务成功率（Task Success Rate, TSR）   正确完成任务的比例   正确数 / 总数
策略复用率（Strategy Reuse Rate, SRR）   使用历史策略的比例   复用策略数 / 总策略数
适应速度（Adaptation Speed, AS）   达到 80% TSR 所需任务数   任务数 @ TSR≥80%
错误率下降率（Error Reduction Rate, ERR）   错误率随经验下降斜率   ΔError / ΔExperience
推理效率（Inference Efficiency, IE）   平均推理步数   总步数 / 任务数

2. 策略质量指标
指标   说明
策略通用性得分（Generality Score）   能在 ≥3 个领域复用的策略占比
策略可解释性（Explainability）   人工评估策略是否可理解（1-5 分）
策略迁移成功率（Transfer Success Rate）   跨任务迁移后性能提升比例

3. 人类对比实验
- 邀请 12 名领域专家（数学、法律、医学等）进行相同任务测试。
- 比较 TSR、AS、IE 三项指标。

五、实现细节与超参数

1. 模型架构
- LLM 基座：GPT-4（API）或 Llama-3-70B（本地部署）
- 图编码器：GraphSAGE + GNN Pooling
- 策略抽象器：基于 图压缩算法（Graph Compression via Modularity Optimization）
- 元认知循环频率：每完成 5 个任务触发一次反思与抽象
- 强化学习优化器：PPO，奖励函数：
  
  R = alpha cdot text{TSR} + beta cdot text{SRR} - gamma cdot text{IE}
  
  （默认 α=0.6, β=0.3, γ=0.1）

2. 训练配置
- 硬件：8× A100 80GB GPU，分布式训练
- 批量大小（Batch Size）：策略图批次为 32
- 学习率：策略选择器：1e-5；图编码器：5e-6
- 训练步数：最多 50,000 步，早停机制（patience=1000）

3. 策略图谱（Strategy Graphbase）
- 容量：初始 100 个策略图，动态增长
- 存储格式：JSON + Neo4j 图数据库
- 检索方式：语义相似度（Sentence-BERT） + 结构匹配（Graph Edit Distance）

4. 元认知循环流程
for task in task_stream:
    result = GoT_Engine.solve(task, strategy_hint=current_strategy)
    if result.success:
        strategy_graph = Meta_Abstractor.extract(result.reasoning_dag)
        Strategy_Graphbase.add(strategy_graph)
        update_policy_network()  # 通过PPO优化

        strategy_graph = Meta_Abstractor.diagnose(result.failure_path)
        Strategy_Graphbase.update(strategy_graph)  # 修正策略

        Strategy_Graphbase.optimize()  # 全局策略压缩与去重

六、实验流程

阶段 1：基线性能测试
- 在 8 个领域分别运行 Meta-GoT 与基线模型
- 记录初始 TSR、IE、AS

阶段 2：自进化能力测试
- 运行 100 个任务序列（混合领域）
- 每 10 个任务评估一次 TSR 与 SRR
- 绘制“学习曲线”：TSR vs 经验数

阶段 3：跨领域迁移测试
- 在数学领域训练 50 个任务
- 在法律领域测试 20 个新任务（无微调）
- 比较迁移前后 TSR 与 AS

阶段 4：消融实验（Ablation Study）
变体   说明
Meta-GoT (w/o 反思)   移除反思模块
Meta-GoT (w/o 抽象)   不提取策略图
Meta-GoT (w/o RL)   无策略优化
Meta-GoT (w/o 图谱)   无策略存储

七、可复现性保障
- 代码开源：Meta-GoT GitHub 仓库，包含：
  - config/：所有超参数配置文件
  - scripts/：实验启动脚本
  - data/：数据预处理脚本
  - eval/：评估指标计算工具
- 日志系统：记录每个任务的：
  - 输入、推理图、策略图、是否成功、耗时、资源消耗
- 随机种子：固定为 42，确保结果可复现

八、伦理与安全考虑
- 所有数据均去标识化，符合 GDPR/CCPA。
- 军事与医疗任务仅用于模拟，不用于真实决策。
- 设置伦理审查模块，自动过滤有害策略建议。
