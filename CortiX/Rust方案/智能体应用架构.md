## 二、关键设计决策

### 1. **通信协议：WebSocket + HTTP 分离**

| 用途 | 协议 | 原因 |
|------|------|------|
| **DSL 执行/流式输出** | WebSocket | AgenticOS 的 Branch 流需要双向实时通信 |
| **大文件传输** | HTTP/2 | 避免 WebSocket 帧分片，支持进度条 |
| **配置/查询** | HTTP REST | 简单请求，缓存友好 |

### 2. **本地服务器发现机制**

浏览器需要知道 Rust 服务器跑在哪个端口：

**方案 A：固定端口（推荐 MVP）**
```rust
// Rust 尝试绑定 3000，被占用则递增
let port = 3000..4000
    .find(|p| TcpListener::bind(("127.0.0.1", *p)).is_ok())
    .expect("No free port");
```

**方案 B：服务发现（高级）**
```rust
// 使用 mDNS 广播 _agentic._tcp.local.
// 浏览器通过 DNS-SD 发现本地服务
```

**前端连接**：
```typescript
// 前端自动发现逻辑
async function discoverBackend(): Promise<string> {
    // 尝试常用端口
    for (const port of [3000, 3001, 3002]) {
        try {
            const resp = await fetch(`http://localhost:${port}/health`, { 
                signal: AbortSignal.timeout(500) 
            });
            if (resp.ok) return `ws://localhost:${port}/ws/agent`;
        } catch {}
    }
    throw new Error("AgenticOS 本地服务未启动");
}
```

### 3. **安全模型：零信任浏览器**

```rust
// Rust 层的安全中间件
async fn security_middleware(req: Request) -> Result<Response, Error> {
    // 1. 仅允许 localhost 来源（防止恶意网页访问）
    let origin = req.headers().get("origin");
    if !origin.map(|o| o.as_bytes().starts_with(b"http://localhost")).unwrap_or(false) {
        return Err(Error::Forbidden);
    }
    
    // 2. 文件路径白名单（防止 ../../../etc/passwd）
    if let Some(path) = req.param("path") {
        validate_path_in_vault(path)?;
    }
    
    // 3. 请求速率限制（防止浏览器插件恶意调用）
    check_rate_limit(&req.ip())?;
    
    Ok(next(req).await)
}
```

### 4. **浏览器 File System Access API（渐进增强）**

标准浏览器无法直接访问文件，但 Chrome/Edge 支持 **File System Access API**：

```typescript
// 请求用户授权访问特定文件夹
async function selectVaultDirectory(): Promise<FileSystemDirectoryHandle> {

    // 保存 handle 到 IndexedDB，下次自动恢复

// 读取文件（直接浏览器操作，不经过 Rust）
async function readNoteDirectly(path: string): Promise<string> {
    const dirHandle = await getSavedDirectoryHandle();
    const fileHandle = await dirHandle.getFileHandle(path);
    const file = await fileHandle.getFile();
    return await file.text();

```

**Fallback 方案**（Firefox/Safari）：

```typescript
    .then(r => r.text());
```

## 三、与 Tauri 方案的详细对比

| 维度 | **Tauri (IPC)** | **Browser + Local Server** |
|------|----------------|---------------------------|
| **架构** | 进程内通信 | 客户端-服务器（本机） |
| **性能** | 极高（内存共享，零拷贝） | 高（localhost TCP，~0.1ms 延迟） |
| **文件系统** | 直接访问（Rust std::fs） | HTTP API 代理或 File System Access API |
| **安装** | 下载安装包（.dmg/.exe） | 访问网页或 PWA 安装 |
| **自动更新** | Tauri 内置 | PWA Service Worker 自动更新 |
| **移动端** | 需单独开发（或 Tauri Mobile） | 直接支持（手机浏览器访问局域网 IP） |
| **离线能力** | 完全离线 | PWA 离线缓存（需提前加载） |
| **调试** | 需编译 Rust | 浏览器 DevTools 直接调试 |
| **部署成本** | 低（单机运行） | 中（需处理 CORS、HTTPS 自签名证书） |

## 四、核心模块架构

### Rust 本地服务器（Axum 实现）

```rust
use axum::{
    routing::{get, post},
    Router,
    extract::{ws::WebSocketUpgrade, State, Path},
    response::IntoResponse,
};
use tower_http::cors::CorsLayer;

#[tokio::main]
async fn main() {
    let state = AppState {
        agentic_os: connect_os("ws://localhost:9001").await,
        vault_path: std::env::var("VAULT_PATH").unwrap(),
    };

    let app = Router::new()
        // WebSocket：浏览器 <-> AgenticOS 的实时流
        .route("/ws/agent", get(agent_websocket_handler))
        
        // HTTP API：文件操作（大文件不走 WebSocket）
        .route("/api/file/*path", get(read_file).post(write_file))
        .route("/api/vector/search", post(vector_search))
        .route("/api/vector/index", post(index_document))
        
        // 健康检查（前端用来发现服务）
        .route("/health", get(|| async { "AgenticOS Gateway Ready" }))
        
        // 静态文件（SPA 支持）
        .fallback_service(tower_http::ServeDir::new("dist").not_found_service(
            tower_http::ServeFile::new("dist/index.html")
        ))
        
        // CORS：严格限制 localhost
        .layer(CorsLayer::new()
            .allow_origin(["http://localhost:3000".parse().unwrap()])
            .allow_methods([Method::GET, Method::POST])
            .allow_headers([header::CONTENT_TYPE])
        )
        .with_state(Arc::new(state));

    let listener = tokio::net::TcpListener::bind("127.0.0.1:3000").await.unwrap();
    axum::serve(listener, app).await.unwrap();
}

// WebSocket 处理：双向桥接
async fn agent_websocket_handler(
    ws: WebSocketUpgrade,
    State(state): State<Arc<AppState>>,
) -> impl IntoResponse {
    ws.on_upgrade(move |socket| handle_socket(socket, state))
}

async fn handle_socket(mut browser_ws: WebSocket, state: Arc<AppState>) {
    let mut os_ws = state.agentic_os.get_connection().await;
    
    loop {
        tokio::select! {
            // 浏览器 -> AgenticOS（用户输入/DSL）
            Some(Ok(msg)) = browser_ws.recv() => {
                if let Ok(text) = msg.to_text() {
                    // 可选：在此处验证 DSL 安全性
                    os_ws.send(Message::Text(text.to_string())).await.ok();
                }
            }
            
            // AgenticOS -> 浏览器（流式结果/Branch 事件）
            Some(Ok(msg)) = os_ws.recv() => {
                browser_ws.send(msg).await.ok();
            }
            
            else => break,
        }
    }
}
```

### 前端架构（PWA）

```typescript
// services/agent-bridge.ts
// 封装浏览器与 Rust 的通信

export class AgentBridge {
    private ws: WebSocket | null = null;
    private reconnectAttempts = 0;
    
    async connect(): Promise<void> {
        const wsUrl = await discoverBackend();
        this.ws = new WebSocket(wsUrl);
        
        this.ws.onmessage = (event) => {
            const msg = JSON.parse(event.data);
            this.handleOSMessage(msg);
        };
        
        // 自动重连逻辑
        this.ws.onclose = () => {
            setTimeout(() => this.connect(), Math.min(1000 * 2 ** this.reconnectAttempts, 30000));
            this.reconnectAttempts++;
        };
    }
    
    // 发送 DSL 执行请求
    executeDSL(dsl: string): EventSource {
        const sessionId = generateUUID();
        this.ws?.send(JSON.stringify({
            type: 'dsl_execute',
            session_id: sessionId,
            dsl,
        }));
        
        // 返回 EventEmitter 监听流式结果
        return this.createEventSource(sessionId);
    }
    
    // 文件操作（HTTP）
    async readFile(path: string): Promise<string> {
        // 优先尝试 File System Access API（Chrome）
        if ('showDirectoryPicker' in window) {
            return await this.readViaFileAPI(path);
        }
        // Fallback 到 Rust 代理
        const resp = await fetch(`http://localhost:3000/api/file/${encodeURIComponent(path)}`);
        return await resp.text();
    }
}
```

```typescript
// 使用 HTTP/2 Server Push 或分片上传
async function uploadLargeFile(file: File) {
    const chunkSize = 1024 * 1024; // 1MB
    for (let start = 0; start < file.size; start += chunkSize) {
        const chunk = file.slice(start, start + chunkSize);
        await fetch('/api/file/upload', {
            method: 'POST',
            headers: {
                'X-File-Name': file.name,
                'X-Chunk-Index': (start / chunkSize).toString(),
            },
            body: chunk,
        });
    }
}
```

**如果前端改为浏览器，这是最优架构**：

```
┌──────────────────────┼──────────────────────────────────────┐
│  Rust Local Server   │ Axum/Actix                           │
│  • 静态文件服务（SPA fallback）                              │
│  • /ws/agent 双向流代理（Browser ↔ AgenticOS）              │
│  • /api/* 文件系统代理（安全路径校验）                        │
│  • 多标签页 Session 管理                                     │
└──────────────────────┬──────────────────────────────────────┘
                       │ WebSocket (AgenticDSL)
┌──────────────────────┼──────────────────────────────────────┐
│  AgenticOS v3        │ 大脑，执行 DSL，管理 Branch/Budget   │
└─────────────────────────────────────────────────────────────┘
```

**相比 Tauri 方案的变化**：
1. **通信层**：IPC → WebSocket/HTTP（延迟从 ~0.01ms 增加到 ~0.1ms，可忽略）
2. **文件访问**：直接 → HTTP API 代理（增加一次序列化，但实现更简单）
3. **部署**：安装包 → 访问 URL（用户门槛更低，但需处理后台服务生命周期）
4. **调试**：浏览器 DevTools 直接调试 Rust 日志（`console.log` 到浏览器 Console）

**唯一注意点**：需要确保 Rust 本地服务器在后台运行（可用 `auto-launch` crate 实现开机自启，或提供"启动守护进程"按钮）。

## 1. 架构宣言（Architecture Manifesto）

本架构遵循**"智能体即应用"（Agent-as-Application）**范式：

- **AgenticOS** 是认知内核（Brain），负责执行 AgenticDSL，管理思维拓扑（Branch/Budget）
- **Rust Gateway** 是具身系统（Embodiment），连接大脑与物理世界（文件系统、网络、传感器）
- **Browser PWA** 是表现层（Avatar），负责渲染 Agent 状态与接收人类输入
- **智能体（Agent）** 是应用本体，具有持续身份、记忆与目标；传统"软件功能"降级为 Agent 的**技能（Skill）**

### 4.1 Layer 1: AgenticOS（认知层）

**职责**：纯粹的思维执行，无状态（Stateless），只处理 DSL。

**接口规范**：
```rust
// AgenticOS 暴露的 WebSocket API
pub trait AgenticOS {
    // 创建会话（分配预算）
    async fn session_create(&self, config: SessionConfig) -> Result<SessionId>;
    
    // 执行 DSL（核心）
    async fn dsl_execute(&self, session: SessionId, dsl: &str) -> Result<Stream<Event>>;
    
    // 思维控制
    async fn fork_branch(&self, session: SessionId, strategy: ForkStrategy) -> Result<BranchId>;
    async fn merge_branches(&self, session: SessionId, branches: Vec<BranchId>) -> Result<Value>;
    
    // 元认知查询
    async fn trace_dag(&self, session: SessionId) -> Result<DAGSnapshot>;
    async fn budget_status(&self, session: SessionId) -> Result<BudgetReport>;
}
```

**AgenticDSL 示例**（PKM 场景）：
```lisp

      (tool :name 'semantic_search

    ;; Branch C: 检查矛盾
    (branch 'consistency_check
      (tool :name 'check_contradiction
            :proposition "所有权系统")))
  
  ;; 3. 合并结论（Join）
  (merge (:strategy 'synthesize)

                  关联：{{branch_b.results}}
                  验证：{{branch_c.contradictions}}"))
  
  ;; 4. 行动：实际创建

       :tool 'note_create

                :content "{{merged_result}}"
                :tags ("rust" "memory" "system")))
  
  ;; 5. 自指：记录本次思考过程到记忆
```

### 4.2 Layer 2: Rust Gateway（具身层）

**职责**：Agent 的躯体，管理生命周期，执行物理动作。

**核心模块**：

#### A. Agent Host（宿主）
```rust
pub struct AgentHost {
    config: HostConfig,
    agent: RwLock<Agent>,                    // 当前运行的 Agent
    skill_registry: Arc<SkillRegistry>,      // 技能市场
    os_client: AgenticOSClient,              // 大脑连接
    browser_sessions: DashMap<SessionId, BrowserConnection>, // 多前端支持
}

impl AgentHost {
    // 启动 Agent（从人格定义初始化）
    pub async fn boot(&self, persona_id: &str) -> Result<()> {
        let persona = load_persona(persona_id)?;
        let os_session = self.os_client.create_session(persona.to_os_config()).await?;
        
        let agent = Agent::new(persona, os_session);
        
        // 加载默认技能
        for skill_id in &persona.default_skills {
            let skill = self.skill_registry.load(skill_id).await?;
            agent.install_skill(skill).await?;
        }
        
        *self.agent.write() = agent;
        self.run_cognition_loop().await;
    }
    
    // 认知循环：桥接 Browser <-> AgenticOS
    async fn run_cognition_loop(&self) {
        loop {
            tokio::select! {
                // 来自浏览器的感知（用户输入）
                Some(percept) = self.browser_rx.recv() => {
                    let dsl = self.agent.read().process_percept(percept);
                    let mut stream = self.os_client.dsl_execute(dsl).await;
                    
                    // 流式转发到浏览器和技能系统
                    while let Some(event) = stream.next().await {
                        self.handle_os_event(event).await;
                    }
                }
                
                // 来自文件系统的感知（自动索引）
                Some(event) = self.fs_watcher.recv() => {
                    let percept = Percept::FileChanged(event);
                    self.agent.read().perceive(percept).await;
                }
            }
        }
    }
}
```

## 6. 安全模型

### 6.1 零信任架构（Zero-Trust）

**Browser ↔ Rust**:
- **Origin 严格限制**: 仅允许 `http://localhost:*` 或配置的 HTTPS 域名
- **CSP (Content Security Policy)**: 禁止内联脚本，仅允许特定 CDN
- **Capability Token**: 每个文件操作需携带时间戳签名 Token（防 CSRF）

**Rust ↔ AgenticOS**:
- **Session 隔离**: 每个 Agent 独立 Session，Budget 硬限制（防资源耗尽）
- **Tool 白名单**: Agent 只能调用已声明 Skill 提供的 Tools（防越权）

**Agent ↔ File System**:
- **路径沙箱**: 所有文件操作限定在 `VAULT_PATH` 内（通过 `chroot` 或容器化）
- **写前确认**: 破坏性操作（删除、覆盖）需用户通过 Browser 确认（防误删）

### 6.2 隐私模型

**本地优先（Local-First）**:
```rust
// 隐私级别配置
enum PrivacyMode {
    Paranoid,    // 所有 LLM 调用使用本地模型（Ollama），零云端传输
    Balanced,    // 本地优先，复杂查询经用户同意后使用云端
    Cloud,       // 使用云端 AgenticOS 集群（需 E2E 加密）
}

impl AgentHost {
    fn ensure_privacy(&self, dsl: &DSL) -> Result<()> {
        if self.config.privacy == PrivacyMode::Paranoid {
            // 检查 DSL 中是否包含 cloud_llm 节点
            if dsl.contains_cloud_services() {
                return Err(Error::PrivacyViolation);
            }
        }
        Ok(())
    }
}
```

## 1. 架构宣言

本架构采用 **"浏览器即容器，智能体即服务"** 范式：

- **AgenticOS**：认知内核（Brain），纯 DSL 执行引擎  
- **Rust Gateway**：本地守护进程（Embodiment），通过 WebSocket/HTTP 暴露能力  
- **Browser PWA**：表现层（Avatar），纯前端应用，通过标准 Web API 与后端通信  
- **Agent**：运行在 Gateway 中的持久化身份，Browser 只是其众多可能的 Avatar 之一  

**关键转变**：从"桌面应用"思维转向"本地 Web 服务"思维——Agent 是常驻本地的服务，浏览器是访问该服务的客户端。

### 3.3 Browser 前端架构（PWA）

```typescript
// 前端 SDK：AgentRuntime for Browser
class BrowserAgentRuntime {
    private ws: WebSocket | null = null;
    private reconnectTimer: number | null = null;
    private sw: ServiceWorker | null = null;
    
    // 自动发现后端（支持 mDNS 或固定端口）
    async discover(): Promise<string> {
        // 尝试常用端口 3000-3010
        for (let port = 3000; port < 3010; port++) {
            try {
                const resp = await fetch(`http://localhost:${port}/health`, {
                    signal: AbortSignal.timeout(500)
                });
                if (resp.ok) return `ws://localhost:${port}/ws`;
            } catch {}
        }
        throw new Error("Agent Gateway not found");
    }
    
    // 连接（自动重连）
    async connect(endpoint?: string) {
        const url = endpoint || await this.discover();
        this.ws = new WebSocket(url);
        
        this.ws.onopen = () => {
            console.log("Connected to Agent");
            this.reconnectTimer && clearTimeout(this.reconnectTimer);
        };
        
        this.ws.onmessage = (e) => {
            const msg = JSON.parse(e.data);
            this.handleAgentEvent(msg); // 驱动 UI 更新
        };
        
        this.ws.onclose = () => {
            // 指数退避重连
            this.reconnectTimer = window.setTimeout(() => this.connect(), 3000);
        };
    }
    
    // 发送感知（用户输入）
    perceive(type: string, content: any) {
        this.ws?.send(JSON.stringify({
            type: 'percept',
            browser_tab_id: this.tabId, // 支持多标签页识别
            percept: { type, content, timestamp: Date.now() }
        }));
    }
    
    // HTTP API：大文件操作（不走 WebSocket）
    async uploadFile(file: File): Promise<string> {
        const form = new FormData();
        form.append('file', file);
        
        const resp = await fetch('http://localhost:3000/api/file/upload', {
            method: 'POST',
            body: form,
            // 使用 Fetch 的 Upload Progress（通过 ReadableStream）
        });
        return resp.json(); // 返回 file_ref
    }
}
```

## 4. 分层详细规范（Browser 版）

### 4.1 Layer 2: Rust Gateway（Web 服务器）

使用 **Axum** 或 **Actix-web** 替代 Tauri，纯 HTTP/WebSocket 服务：

```rust
use axum::{
    routing::{get, post},
    Router,
    extract::{ws::WebSocketUpgrade, State, Path},
    response::IntoResponse,
    http::Method,
};
use tower_http::cors::{CorsLayer, AllowOrigin};

#[tokio::main]
async fn main() {
    let state = Arc::new(AgentHost::new().await);
    
    let app = Router::new()
        // WebSocket: 浏览器实时连接（替代 Tauri IPC）
        .route("/ws", get(websocket_handler))
        
        // HTTP API: 文件系统代理（替代直接 fs 访问）
        .route("/api/file/:path", get(read_file).post(write_file))
        .route("/api/file/upload", post(upload_file))
        
        // 向量查询
        .route("/api/vector/search", post(vector_search))
        .route("/api/vector/index", post(index_document))
        
        // Agent 控制
        .route("/api/agent/state", get(get_agent_state))
        .route("/api/agent/act", post(trigger_action))
        
        // 健康检查（用于浏览器发现服务）
        .route("/health", get(|| async { "Agent Gateway Ready" }))
        
        // CORS: 严格限制 localhost（安全）
        .layer(
            CorsLayer::new()
                .allow_origin(AllowOrigin::predicate(|origin, _| {
                    origin.as_bytes().starts_with(b"http://localhost") ||
                    origin.as_bytes().starts_with(b"https://localhost")
                }))
                .allow_methods([Method::GET, Method::POST, Method::DELETE])
                .allow_headers([header::CONTENT_TYPE, header::AUTHORIZATION])
        )
        .with_state(state);

    let listener = tokio::net::TcpListener::bind("127.0.0.1:3000").await.unwrap();
    println!("Agent Gateway listening on http://127.0.0.1:3000");
    axum::serve(listener, app).await.unwrap();
}

// WebSocket 处理：桥接 Browser ↔ AgenticOS
async fn websocket_handler(
    ws: WebSocketUpgrade,
    State(state): State<Arc<AgentHost>>,
) -> impl IntoResponse {
    ws.on_upgrade(move |socket| handle_browser_socket(socket, state))
}

async fn handle_browser_socket(mut browser_ws: WebSocket, state: Arc<AgentHost>) {
    // 生成浏览器标签页 ID
    let tab_id = generate_tab_id();
    
    // 注册到 Agent（支持多前端）
    let agent = state.current_agent();
    agent.register_browser_tab(tab_id, browser_ws.clone()).await;
    
    // 双向桥接
    loop {
        tokio::select! {
            // 浏览器 → AgenticOS（用户输入）
            Some(Ok(msg)) = browser_ws.recv() => {
                if let Ok(text) = msg.to_text() {
                    let percept: Percept = serde_json::from_str(text).unwrap();
                    // 触发 Agent 认知循环
                    agent.perceive(percept).await;
                }
            }
            
            // AgenticOS → 浏览器（流式结果）
            Some(event) = agent.outbound_rx.recv() => {
                browser_ws.send(Message::Text(serde_json::to_string(&event).unwrap())).await.ok();
            }
        }
    }
}
```

```rust
    Path(path): Path<String>,
    State(state): State<Arc<AgentHost>>,
```

## 5. 安全模型（Browser 适配）

### 5.1 浏览器 ↔ Rust Gateway 安全

| 威胁 | 防护措施 |
|------|---------|
| **CSRF** | Origin 严格限制 `localhost:*`，Token 时间戳签名 |
| **XSS** | CSP 策略：`script-src 'self'`；技能代码在沙箱 iframe |
| **路径遍历** | Rust 层 `sanitize_path`，限定 `VAULT_PATH` 沙箱 |
| **DoS** | 请求速率限制（Tower layer），WebSocket 消息大小限制 |
| **窃听** | WebSocket 可选 TLS（wss://），即使 localhost |

### 5.2 多设备访问控制（Home Server 模式）

当 Rust Gateway 绑定到 `0.0.0.0`（局域网共享）：
```rust
// 设备认证（首次配对）
async fn pair_device(headers: HeaderMap) -> Result<Token, Error> {

        return Err(Error::Forbidden);
    }
```

- 通信：Gateway 通过 TLS WebSocket 连接云端 OS
- Browser: 连接本地 Gateway（隐私优先）

## 8. Browser 特有优势与限制

### 优势
1. **零安装门槛**：访问 URL 即用，降低用户试用成本
2. **真·跨平台**：同一 URL 在手机、平板、PC 上自适应
3. **开发速度**：Vite HMR + 浏览器 DevTools，迭代极快
4. **生态复用**：可直接嵌入 CodeSandbox、Excalidraw 等 Web 组件

### 限制与应对
1. **文件系统受限**  
   → 使用 File System Access API（Chrome）或 Rust 代理层（Fallback）

2. **后台运行受限**  
   → Service Worker + Background Sync，或引导用户安装 PWA 并常驻标签页

3. **系统快捷键冲突**（如 Ctrl+W 关闭标签）  
   → 使用 `beforeunload` 提示，或引导用户安装为桌面 PWA（窗口模式）

4. **WebView 性能差异**  
   → 复杂图谱渲染使用 OffscreenCanvas，避开主线程
