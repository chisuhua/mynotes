让 AI 在回答时**优先引用新文献**，不能仅靠“口头嘱咐”，必须通过 **“检索层加权” + “提示词强约束” + “重排序机制”** 三位一体的策略来实现。

在 OpenClaw 架构下，以下是具体的配置方案：

---

### 策略一：检索层加权 (RAG Retrieval Weighting) —— 最根本
**原理**：在向量数据库检索阶段，直接给新文献更高的**分数权重**。这样即使旧文献语义匹配度稍高，新文献也会因为“时间加分”而排在前面，优先进入 AI 的上下文窗口。

#### 1. 配置 `openclaw.json` (混合检索 + 时间衰减)
启用支持元数据过滤和评分函数的向量数据库（如 Chroma, Qdrant, 或 OpenClaw 内置引擎）。

```json
{
  "retrieval": {
    "strategy": "hybrid_time_aware",
    "config": {
      "base_top_k": 10,
      "time_decay_factor": 0.1, 
      "recency_bonus": {
        "enabled": true,
        "rules": [
          { "condition": "year >= 2024", "score_boost": 0.5 },
          { "condition": "year >= 2022", "score_boost": 0.2 },
          { "condition": "year < 2020", "score_penalty": -0.3 }
        ]
      },
      "filters": {
        "must": [
          { "key": "status", "operator": "eq", "value": "active" }
        ]
      }
    }
  }
}
```
*   **效果**：一篇 2024 年的论文，即使语义相似度是 0.75，加上 0.5 的 bonus 后变成 1.25，会稳稳排在相似度 0.85 但年份为 2018 的论文之前。

#### 2. 动态时间窗口过滤
在用户提问涉及“最新进展”、“现状”、“趋势”等关键词时，自动收紧检索时间范围。

*   **OpenClaw Router 逻辑**：
    ```python
    if user_query contains ["最新", "进展", "现状", "2024", "2025", "recent"]:
        retrieval_config.filters.append({"key": "year", "operator": "gte", "value": 2023})
    ```

---

### 策略二：提示词强约束 (Prompt Engineering) —— 最直接
**原理**：在 System Prompt 中明确指令，要求 AI 对检索到的文档进行**二次筛选**，并强制规定引用优先级。

#### 1. 修改 Agent 的 System Prompt
在你的 `researcher` 或 `chat-agent` 配置中加入以下模块：

```markdown
# 📅 文献时效性优先协议 (Recency Priority Protocol)

你正在回答一个学术问题。请严格遵守以下引用规则：

1. **优先级排序**：
   - **第一梯队 (必须优先)**：近 2 年 (2024-2026) 的文献。只要存在相关新文献，**严禁**只引用旧文献。
   - **第二梯队 (补充参考)**：3-5 年前 (2021-2023) 的文献。仅当新文献不足时使用。
   - **第三梯队 (历史背景)**：5 年前 (<2021) 的文献。仅用于介绍基础理论或历史演变，**不得**作为当前技术状态的证据。

2. **冲突裁决**：
   - 如果新旧文献结论冲突，**必须以新文献为准**。
   - 话术示例：“虽然早期研究 (Author, 2018) 认为 X，但最新实证 (Author, 2025) 表明 Y 才是主流...”

3. **显式声明**：
   - 在回答开头，必须简要说明引用的时间范围。例如：“基于 2023-2026 年的最新研究...”
   - 如果被迫引用了超过 5 年的文献，必须加注："⚠️ 注：该观点源自较早研究，需结合最新进展审视。”

4. **无新文献处理**：
   - 如果检索结果中缺乏近 3 年文献，请直接告知用户：“目前库中关于此主题的最新研究仅更新至 XXXX 年，以下为基于现有资料的分析..." **切勿编造不存在的“最新”文献。**
```

#### 2. Few-Shot 示例 (增强理解)
在 Prompt 中提供正反例：

> **用户**：LLM 量化的最新方法有哪些？
>
> ❌ **错误回答**：Gong et al. (2018) 提出了量化感知训练... (忽略了 2024 年的新方法)
>
> ✅ **正确回答**：根据 **2024-2025 年** 的最新研究，主流方法已转向稀疏化量化。例如，Smith et al. (2025) 提出了... 而早期的 Gong et al. (2018) 方法虽奠定了基础，但在大模型场景下已不再适用。

### 策略三：重排序机制 (Re-Ranking) —— 最精准
**原理**：在检索出 Top-20 文档后，使用一个专门的 **Re-Ranker 模型** 或 **LLM 自身**，根据“相关性 + 时效性”重新排序，只把前 5 个（主要是新的）送给生成模型。

#### 1. 启用 Re-Ranker (如果 OpenClaw 支持)
配置 Cross-Encoder 模型，并在评分函数中加入时间因子。
```json
"reranker": {
  "model": "bge-reranker-v2-m3",
  "custom_scoring_formula": "0.7 * relevance_score + 0.3 * time_score"
}
```

#### 2. LLM 自我重排序 (Self-Re-Rank)
如果不方便部署重排模型，可以让 AI 在生成前先做一次“思维链”排序：

> **Prompt 指令**：
> “在生成最终回答前，请先执行以下步骤：
> 1. 列出检索到的所有候选文献及其年份。
> 2. 按年份从新到旧排序。
> 3. 剔除那些虽然相关但已被更新研究推翻的旧文献。
> 4. 仅选取排序前 5 名的文献作为核心依据生成回答。”

---

### 策略四：用户侧引导 (UI/UX)
在前端界面或默认提问模板中，帮用户加上时间约束。

*   **默认追问建议**：当用户问完一个问题，AI 回答后，底部按钮显示：
    *   “🔍 只看 2024 年以后的研究”
    *   “🔄 对比新旧观点差异”
*   **自动追加指令**：在发送请求给后端前，JS 自动在 query 后追加 `--min-year 2023` (如果系统支持这种语法糖)。

### 综合实战配置示例

将上述策略整合到 OpenClaw 的一个完整调用流程中：

**1. 用户提问**：
> "Transformer 架构有什么最新改进？"

**2. OpenClaw 处理流程**：
*   **Step 1 (路由识别)**：检测到“最新改进”，触发 `time_filter: year >= 2023`。
*   **Step 2 (检索)**：向量检索召回 20 篇文档。
*   **Step 3 (加权)**：应用 `recency_bonus`，2024-2026 的文档得分大幅提升，占据 Top 10 中的 8 席。
*   **Step 4 (重排序)**：Re-Ranker 再次确认，确保最相关的**新**文献排在最前。
*   **Step 5 (生成)**：LLM 收到包含 8 篇新文献的 Context，并根据 System Prompt 中的“优先协议”，生成如下回答：

> **AI 回答**：
> “基于 **2023-2026 年** 的最新文献，Transformer 架构的改进主要集中在效率优化和长上下文处理上：
> 1. **线性注意力机制**：最新的 **Zhang et al. (2025)** 提出了... (取代了早期的近似方法)。
> 2. **混合专家模型 (MoE)**：**Google (2024)** 的最新报告显示...
>
> *注：早期的 Vaswani et al. (2017) 奠定了基础，但上述新研究已解决了其计算复杂度高的问题。*"

### 总结
要让 AI 优先引用新文献，**不要只依赖 AI 的“自觉”**，必须：
1.  **物理上**：在检索阶段给新文献**加分**、给旧文献**设限**。
2.  **逻辑上**：在 Prompt 中规定**“有新必用新，无新要说明”**的硬性规则。
3.  **流程上**：引入**重排序**，确保新文献进入上下文窗口的核心位置。

这样配置后，你的 AI 就会像一个时刻关注前沿的科研助手，而不是一个抱着老黄历不放的图书管理员。
