# KnowledgeGraph 核心需求与功能定义

**版本**: v1.0  
**最后更新**: 2026-03-23  
**状态**: 架构评审中

---

## 🎯 核心目标

构建一个**基于 OpenClaw 的学术文献知识图谱工具**，实现：

1. **自动文献获取** — 7x24 小时监控 arXiv/学术数据库，自动下载相关论文
2. **智能知识抽取** — 从 PDF 中自动提取实体、关系、SOTA 指标
3. **演进分析** — 识别"谁超越了谁"，构建 SOTA 演进链
4. **冲突检测** — 自动发现结论冲突，标记过期文献
5. **静态展示** — MkDocs 生成可导航的文献知识库网站

---

## 📦 核心功能模块

### 1. 文献获取模块

| 功能 | 描述 | 实现方式 |
|---|---|---|
| 自动监控 | 按关键词监控 arXiv/Google Scholar | `paper-hunter` Agent |
| 智能筛选 | 相关性评分 > 0.8 才下载 | LLM 判断摘要相关性 |
| 去重检查 | 标题/DOI 哈希比对 | 本地索引比对 |
| 分层获取 | 优先预印本/OA，次选机构代理 | Unpaywall API + Proxy |
| 待办队列 | 付费墙文献加入待办列表 | 邮件通知用户 |

### 2. PDF 解析模块

| 功能 | 描述 | 实现方式 |
|---|---|---|
| 文本提取 | 双栏论文、公式、参考文献 | PyMuPDF / Grobid |
| 表格提取 | 跨页表格、复杂格式 | Camelot / LLM 视觉提取 |
| 图表提取 | 图表描述生成 | Qwen-VL 多模态 |
| 元数据提取 | 标题/作者/DOI/年份 | 结构化解析 |

### 3. 知识抽取模块

| 功能 | 描述 | 实现方式 |
|---|---|---|
| 实体识别 | Paper/Author/Method/Dataset/Metric | LLM + Prompt |
| 关系识别 | CITES/USES_DATASET/PROPOSES_METHOD | LLM + Prompt |
| SOTA 关系 | SURPASSES (超越关系) | LLM 识别比较性陈述 |
| 冲突识别 | REFUTES (反驳关系) | LLM 识别否定性陈述 |

### 4. 存储层模块

| 功能 | 描述 | 实现方式 |
|---|---|---|
| 图数据存储 | 节点/关系/属性 | UniDAG-Store 扩展 |
| 图查询接口 | 邻居查询/路径查询/Cypher 子集 | UniDAG-Adapter |
| 增量更新 | 新文献入库不重建全量 | 增量写入 |
| 事务保证 | 抽取失败可回滚 | 事务日志 |

### 5. 分析模块

| 功能 | 描述 | 实现方式 |
|---|---|---|
| SOTA 演进链 | 时间线上的 SOTA 更替 | 遍历 SURPASSES 关系 |
| 冲突检测 | 新旧文献结论对比 | LLM 裁决 |
| 趋势分析 | 技术路线分叉识别 | DAG 主干/旁支分析 |
| 过期扫描 | 定期扫描旧 SOTA 文献 | Cron + 联网验证 |

### 6. 展示模块

| 功能 | 描述 | 实现方式 |
|---|---|---|
| MkDocs 生成 | 静态站点构建 | MkDocs + Material |
| 增量更新 | 新文献触发增量生成 | 文件监听 |
| 导航索引 | 分类/时间/作者索引 | 自动生成 |
| 警告注入 | 过期文献顶部警告框 | Frontmatter 标记 |

## 🔧 工具集与 Skill 集划分

### 工具集 (Tools) — 原子操作

```
Tools/
├── pdf-extract          # PDF → Markdown + 元数据
├── table-extract        # 表格 → CSV/JSON
├── figure-extract       # 图表 → 描述文本
├── entity-extract       # LLM → 实体识别
├── relation-extract     # LLM → 关系识别
├── sota-extract         # LLM → 超越关系识别
├── conflict-detect      # 结论冲突检测
├── sota-evolution       # SOTA 演进链分析
├── trend-analysis       # 技术趋势分析
└── mkdocs-generate      # 增量生成静态站点
```

### Skill 集 (Skills) — 工作流编排

```
Skills/
├── auto-ingest-pipeline     # 主流程：监控→下载→解析→抽取→入库→展示
├── paper-hunter-skill       # 文献搜索与下载编排
├── knowledge-extract-skill  # 编排 pdf-extract + entity-extract + relation-extract
├── sota-analysis-skill      # 编排 sota-extract + sota-evolution
├── conflict-scan-skill      # 编排 conflict-detect + 警告注入
├── obsolescence-scan-skill  # 定期过期扫描编排
└── site-update-skill        # MkDocs 增量更新编排
```

### Agent 集 (Agents) — 智能体

```
Agents/
├── paper-hunter       # 学术文献猎手 (搜索/筛选/下载)
├── knowledge-curator  # 知识策展人 (抽取/校验)
└── sota-analyst      # SOTA 分析师 (演进/趋势)
```

## ⚙️ 非功能需求

| 需求 | 描述 |
|---|---|
| **OpenClaw 原生** | 基于 Skill/Tool/Agent 架构，不重复造轮子 |
| **存储统一** | 与 UniDAG-Store 统一设计，不引入 Neo4j 等外部依赖 |
| **增量更新** | 支持触发式更新，避免全量重建 |
| **多 Agent 协作** | 利用 OpenClaw 原生多阶段/多 Agent 能力 |
| **合规获取** | 优先 OA/预印本，付费墙文献加入待办而非暴力破解 |
| **可追溯** | 所有自动标记需记录原因和来源文献 |
