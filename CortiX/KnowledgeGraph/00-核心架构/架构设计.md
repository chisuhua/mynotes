## 🔧 工具集与 Skill 集详细设计

### 工具集 (Tools) — 原子操作

**原则**：每个 Tool 只做一件事，可被多个 Skill 复用

| Tool 名称 | 输入 | 输出 | 描述 |
|---|---|---|---|
| `pdf-extract` | PDF 路径 | Markdown + 元数据 JSON | 文本/公式/参考文献提取 |
| `table-extract` | PDF 路径 + 页码 | CSV/JSON | 表格提取（支持跨页） |
| `figure-extract` | PDF 路径 + 页码 | 图片 + 描述文本 | 图表提取 + 多模态描述 |
| `entity-extract` | Markdown 文本 | 实体列表 JSON | LLM 识别 Paper/Author/Method 等 |
| `relation-extract` | Markdown + 实体列表 | 关系列表 JSON | LLM 识别 CITES/USES_DATASET 等 |
| `sota-extract` | Markdown + 实体列表 | SURPASSES 关系 JSON | 识别"超越"关系 |
| `conflict-detect` | 新文献 Claim + 旧文献库 | 冲突报告 JSON | LLM 裁决结论冲突 |
| `sota-evolution` | 图谱数据 | SOTA 演进链 JSON | 遍历 SURPASSES 构建时间线 |
| `trend-analysis` | 图谱数据 | 技术趋势报告 | DAG 主干/旁支分析 |
| `mkdocs-generate` | 图谱数据 + 模板 | MkDocs 站点文件 | 增量生成静态站点 |

### Skill 集 (Skills) — 工作流编排

**原则**：每个 Skill 编排多个 Tool，实现完整业务流

| Skill 名称 | 编排的 Tools | 触发方式 | 描述 |
|---|---|---|---|
| `auto-ingest-pipeline` | pdf-extract → entity-extract → relation-extract → sota-extract → mkdocs-generate | 文件监听/Cron | 主流程：监控→下载→解析→抽取→入库→展示 |
| `paper-hunter-skill` | (调用 paper-hunter Agent) | Cron (每日) | 文献搜索与下载编排 |
| `knowledge-extract-skill` | pdf-extract + entity-extract + relation-extract | 手动/自动 | 知识抽取编排 |
| `sota-analysis-skill` | sota-extract + sota-evolution | 手动/入库后 | SOTA 分析编排 |
| `conflict-scan-skill` | conflict-detect + 警告注入 | 入库后自动 | 冲突检测与标记编排 |
| `obsolescence-scan-skill` | (联网验证) + 标记 | Cron (每周) | 定期过期扫描编排 |
| `site-update-skill` | mkdocs-generate | 触发式 | MkDocs 增量更新编排 |

### Agent 集 (Agents) — 智能体

**原则**：Agent 带 System Prompt，可自主决策

| Agent 名称 | System Prompt 要点 | 调用的 Tools |
|---|---|---|
| `paper-hunter` | 监控 arXiv/Google Scholar，仅抓取最近 7 天、相关性>0.8 的论文 | web_search, arxiv_api, file_download, duplicate_checker |
| `knowledge-curator` | 从 PDF 中抽取实体/关系，校验一致性 | entity-extract, relation-extract, sota-extract |
| `sota-analyst` | 分析 SOTA 演进链，识别技术趋势 | sota-evolution, trend-analysis |

## 🗄️ 存储层设计 (与 UniDAG-Store 统一)

### 方案 A：UniDAG-Store 扩展图查询（渐进式）

**阶段 1：基础存储**
- UniDAG 存储节点（Paper/Author/Method 等）
- UniDAG 存储关系（CITES/SURPASSES 等）
- 基础查询：邻居查询、单跳关系

**阶段 2：图查询接口**
- 实现 Cypher 子集（MATCH/WHERE/RETURN）
- 路径查询（最短路径、所有路径）
- 反向索引加速关系查询

**阶段 3：高级分析**
- SOTA 演进链查询（时间线遍历）
- 冲突检测查询（结论相似度比对）
- 趋势分析查询（DAG 主干识别）

### Schema 设计（核心实体与关系）

**实体类型 (Nodes)**:
| 实体 | 核心属性 |
|---|---|
| `Paper` | title, year, doi, venue, pdf_url, abstract, status (active/deprecated) |
| `Author` | name, affiliation, orcid_id |
| `Task` | name (如 "Image Classification") |
| `Dataset` | name, domain, url |
| `Metric` | name (如 "Accuracy"), direction (higher_is_better) |
| `Method` | name (如 "Transformer"), category |

**关系类型 (Edges)**:
| 关系 | 方向 | 属性 |
|---|---|---|
| `WRITTEN_BY` | Paper → Author | order (第几作者) |
| `CITES` | Paper → Paper | context_snippet, count |
| `SURPASSES` | Paper → Paper | metric_name, improvement_delta, dataset |
| `REFUTES` | Paper → Paper | reason, confidence |
| `PROPOSES_METHOD` | Paper → Method | - |
| `USES_DATASET` | Paper → Dataset | split (train/test) |
| `ACHIEVES_METRIC` | Paper → Metric | value (数值), dataset |

## ⚡ 触发机制设计

| 触发场景 | 触发方式 | 执行的 Skills | 更新内容 |
|---|---|---|---|
| **新文献入库** | 文件监听 (Watchdog) / Git Hook | auto-ingest-pipeline | 解析→抽取→入库→MkDocs 增量 |
| **手动触发抽取** | 用户命令 (`openclaw run-skill`) | knowledge-extract-skill | 对指定文献重新抽取/分析 |
| **新文献入库后** | 自动触发 | conflict-scan-skill, sota-analysis-skill | 冲突检测 + SOTA 分析 |
| **定期过期扫描** | Cron (每周) | obsolescence-scan-skill | 扫描旧文献→标记过期→MkDocs 增量 |
| **手动更新站点** | 用户命令 | site-update-skill | MkDocs 全量/增量重建 |

## 🌐 与 CortiX 其他子项目关系

```
CortiX (总项目)
├── BrainSkillForge (AgenticDSL 运行时) — 独立，KnowledgeGraph 不直接使用
├── UniDAG-Store (存储层) ← KnowledgeGraph 存储层统一设计
├── Hydra-SKILL (MLA 推理) — 独立，KnowledgeGraph 不直接协作
├── Synapse-SKILL (多智能体) — 独立，KnowledgeGraph 用 OpenClaw 原生 Agent
└── KnowledgeGraph (本工具) — 基于 OpenClaw 现有能力，仅存储层统一
```

**集成边界**：
- **存储层**：通过 UniDAG-Adapter 抽象，未来可无缝切换到其他 CortiX 存储
- **其他层**：完全基于 OpenClaw 现有能力，不依赖 CortiX 其他模块
