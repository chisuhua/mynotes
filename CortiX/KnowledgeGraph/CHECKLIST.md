# 评审检查清单 — KnowledgeGraph

> **使用说明**: 
> - **通用检查项**适用于所有项目，重点关注架构层面的并发安全、资源管理、性能
> - **项目特定检查项**根据技术栈定制（如 CUDA、C++、Rust 等）
> - 评审时逐项勾选，输出表格时引用此项

---

## 通用检查项（所有项目）

### 🔴 Critical（必须修复）

#### 并发安全
- [ ] 多线程/多进程竞争条件
- [ ] 内存序/原子性问题（C++ memory_order、std::atomic）
- [ ] 死锁风险（循环等待、资源抢占、锁顺序不一致）
- [ ] 共享状态未加锁或锁粒度不当
- [ ] 条件变量虚假唤醒处理

#### 资源泄漏
- [ ] 内存泄漏（malloc/new 未 free/delete）
- [ ] 文件句柄泄漏（fopen 未 fclose、open 未 close）
- [ ] 网络连接未关闭（socket、HTTP 连接）
- [ ] GPU 显存未释放（cudaMalloc/cuCtxCreate 未配对释放）
- [ ] 数据库连接未释放
- [ ] 锁未释放（pthread_mutex_lock 未 unlock）

#### 安全漏洞
- [ ] 硬编码密钥/密码/API Token
- [ ] SQL 注入风险
- [ ] 路径遍历漏洞
- [ ] 未验证的用户输入
- [ ] 缓冲区溢出风险

---

### 🟡 Warning（建议修复）

#### 性能陷阱
- [ ] 不必要的内存拷贝（可改用移动语义或引用）
- [ ] 低效的数据结构选择（如用 vector 频繁插入）
- [ ] 同步原语滥用（锁粒度过大、自旋锁滥用）
- [ ] I/O 瓶颈（同步 I/O、小文件频繁读写）
- [ ] 循环内分配内存
- [ ] 未使用缓存/池化

#### 可维护性
- [ ] 魔法数字（未命名的常量）
- [ ] 过深的嵌套（>4 层）
- [ ] 缺失的注释（复杂逻辑、算法、边界条件）
- [ ] 不一致的命名规范
- [ ] 过长的函数（>50 行）
- [ ] 重复代码（违反 DRY）

#### 错误处理
- [ ] 未处理的异常
- [ ] 错误码未检查
- [ ] 无意义的错误消息
- [ ] 异常吞没（catch 后无操作）

---

### 🟢 Info（可选优化）

- [ ] 代码重复（DRY 原则）
- [ ] 日志级别是否合适（DEBUG/INFO/WARN/ERROR）
- [ ] 单元测试覆盖率（目标 >80%）
- [ ] 文档完整性（README、API 文档）
- [ ] 代码格式统一（clang-format、prettier）
