## 🎯 抽取目标

从 PDF 文献中自动提取：
1. **实体** — Paper, Author, Method, Dataset, Metric
2. **关系** — CITES, SURPASSES, REFUTES, USES_DATASET, ACHIEVES_METRIC
3. **SOTA 指标** — 具体数值、数据集、对比基线

---

## 📋 抽取流程

```
PDF → 文本提取 → 段落分割 → 实体识别 → 关系识别 → SOTA 抽取 → 校验 → 入库
```

### 阶段 1: 文本提取
- 使用 `pdf-extract` Tool (PyMuPDF/Grobid)
- 输出：Markdown + 元数据 JSON

### 阶段 2: 实体识别
- 使用 `entity-extract` Tool (LLM Prompt)
- 输入：Markdown 文本
- 输出：实体列表 JSON

**Prompt 示例**:
```
从以下论文片段中识别以下实体：
- Paper (标题、年份、DOI)
- Author (姓名、机构)
- Method (方法名称、类别)
- Dataset (数据集名称)
- Metric (指标名称、方向)

文本：{text_snippet}
```

### 阶段 3: 关系识别
- 使用 `relation-extract` Tool (LLM Prompt)
- 输入：Markdown + 实体列表
- 输出：关系列表 JSON

**Prompt 示例**:
```
基于以下实体和文本，识别以下关系：
- CITES (引用)
- SURPASSES (超越)
- REFUTES (反驳)
- USES_DATASET (使用数据集)
- ACHIEVES_METRIC (达到指标)

实体：{entities_json}
文本：{text_snippet}
```

### 阶段 4: SOTA 关系抽取
- 使用 `sota-extract` Tool (LLM Prompt)
- 重点识别比较性陈述

**关键模式**:
- "Our method achieves X% accuracy, surpassing the previous SOTA [Ref] by Y%."
- "Unlike [Ref], our approach solves the limitation of..."
- "We set a new record on [Dataset] compared to [Ref]."

**输出**:
```json
{
  "source": "LLaMA-3-2024",
  "target": "Mistral-7B-2023",
  "relation": "surpasses",
  "metric": "MMLU Score",
  "value_change": "+5.2%",
  "dataset": "MMLU-57"
}
```

### 阶段 5: 校验
- 检查实体一致性（DOI 唯一性）
- 检查关系合理性（SURPASSES 的 value_delta 应为正数）
- LLM 二次校验（可选）

## ⚠️ 常见问题与处理

| 问题 | 原因 | 缓解措施 |
|---|---|---|
| 实体识别错误 | LLM 理解偏差 | Few-Shot 示例增强，Prompt 迭代 |
| 关系抽取遗漏 | 文本表述隐晦 | 增加上下文窗口，多轮抽取 |
| SOTA 数值错误 | 表格解析失败 | 优先从 Table 实体提取，而非文本 |
| 冲突检测误判 | LLM 裁决不准 | 增加置信度阈值，低置信度转人工 |
