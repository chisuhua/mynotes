## 🎯 核心目标

构建**自动文献获取流水线**，实现：
- 7x24 小时监控 arXiv/Google Scholar
- 按关键词自动筛选相关性 > 0.8 的论文
- 分层获取（优先 OA/预印本）
- 去重检查，避免重复下载

## 🏗️ 架构：文献猎手 Agent

### Agent 定义

```yaml
# agents/paper-hunter.yaml
id: paper-hunter
name: 学术文献猎手
role: 监控 arXiv/Google Scholar，自动发现、下载并预处理最新论文

systemPrompt: |
  你是一个专业的学术文献猎手。你的任务是：
  1. 根据用户设定的关键词监控 arXiv 或特定数据库
  2. 仅抓取最近 7 天内发布的新论文
  3. 阅读摘要，判断是否与主题高度相关（相关性 > 0.8）
  4. 下载 PDF 到本地指定文件夹
  5. 生成初步的元数据（标题、作者、摘要、URL、PDF 路径）

  ⚠️ 注意：
  - 不要下载重复文献（检查标题哈希）
  - 优先下载开源 PDF (arXiv)，避免付费墙
  - 如果找不到 PDF，记录 URL 并标记为 'pending_download'

tools:
  - web_search
  - arxiv_api
  - file_download
  - pdf_parser
  - duplicate_checker
```

## 📋 获取优先级（分层策略）

| 优先级 | 渠道 | 描述 | 风险 |
|---|---|---|---|
| **Level 1** | 开放获取 (OA) | 直接下载 | 🟢 低 |
| **Level 2** | 预印本服务器 | arXiv, bioRxiv, SSRN | 🟢 低 |
| **Level 3** | 机构代理/VPN | 利用学校/公司订阅权限 | 🟡 中 (需防封) |
| **Level 4** | Unpaywall API | 自动寻找合法免费版本 | 🟢 低 |
| **Level 5** | 邮件自动请求 | 给作者发邮件索取 | 🟢 低 |
| **Level 6** | 手动下载 | 加入待办列表，用户手动处理 | 🟢 低 |

## 🔄 自动流水线 Skill

### auto-ingest-pipeline Skill

```markdown
# Skill: auto-ingest-pipeline
# 描述：全自动文献获取与入库流程

## 工作流
1. 设定监控主题：读取配置文件 `config/topics.json` 中的关键词列表
2. 执行搜索：调用 `paper-hunter` Agent，对每个关键词执行 arXiv 搜索（时间范围：最近 7 天）
3. 去重过滤：对比本地数据库中已有的文献标题，剔除已存在的文献
4. 批量下载：对新文献，下载 PDF 到 `~/incoming-papers/`，记录下载日志
5. 深度解析：调用 `batch-paper-tagger` 技能，对下载的 PDF 进行全文提取、标签生成、分类、摘要总结
6. 冲突检测：运行 `conflict-detector`，检查新文献是否推翻了库中旧文献的观点
7. 重建索引：运行 `update-index` 刷新导航页
8. 生成日报：输出 `reports/daily_ingest_YYYY-MM-DD.md`

## 输入参数
- `topics`: 监控关键词列表 (默认读取 config)
- `days_back`: 搜索最近 N 天的文献 (默认 7)
- `min_relevance`: 最低相关性阈值 (默认 0.8)

## 输出
- 新增的 Markdown 文献文件
- 更新的导航索引
- 每日入库报告
```

```bash
0 2 * * * cd /workspace && openclaw run-skill auto-ingest-pipeline >> logs/auto_ingest.log 2>&1
```

### GitHub Actions（云端部署）

```yaml
# .github/workflows/daily-ingest.yml
name: Daily Paper Ingest

on:
  schedule:
    - cron: '0 2 * * *'
  workflow_dispatch:

jobs:
  ingest:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Setup Python & OpenClaw
        run: pip install openclaw arxiv
      - name: Run Auto Ingest Pipeline
        run: openclaw run-skill auto-ingest-pipeline
      - name: Commit and Push Changes
        run: |
          git add .
          git commit -m "docs: 自动入库 $(date +%Y-%m-%d)"
          git push
```

## 🔍 去重检查

```python
# check_title_hash.py
import hashlib
import json
import os

def check(title, db_path):
    title_hash = hashlib.md5(title.lower().encode()).hexdigest()
    index_file = os.path.join(db_path, ".index.json")
    
    if not os.path.exists(index_file):
        return False  # 库为空，不重复
    
    with open(index_file, 'r') as f:
        existing_hashes = json.load(f).get('title_hashes', [])
    
    return title_hash in existing_hashes
```

## ⚠️ 合规注意事项

1. **版权风险** — 绕过付费墙下载受版权保护的论文可能违反版权法
2. **机构封禁** — 大规模自动下载可能导致 IP 被封禁
3. **建议** — 优先 Level 1-4（合法渠道），Level 5-6 加入待办由用户手动处理
