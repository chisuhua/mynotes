# 基于 AgenticDSL 的并行架构与 PromptC 的两阶段编译理念，我为你设计一套**渐进式、路径中心化**的整合方案：

---

## 核心架构：双语法体系 + 路径即类型

### 1. 语法分层定位

| 语法层 | 格式 | 使用者 | 生命周期 | 关键特性 |
|--------|------|--------|----------|----------|
| **PromptC Source** | 类 Python 声明式 | 人类开发者 | 编译期 → IR | 类型推断、沙箱注解、模式匹配 |
| **PromptC-IR** | JSON/YAML（兼容 AgenticDSL） | LLM 生成 / 运行时执行 / 编译器输出 | 运行时 JIT / 持久化 | 路径唯一标识、COW 上下文、预算控制 |

**关键设计**：PromptC-IR 不仅是中间表示，更是**运行时动态图的契约格式**，支持从 `/dynamic` 到 `/lib` 的渐进式固化。

---

## 2. 路径即类型系统（Namespace-as-Type）

借鉴 AgenticDSL 的路径隔离，PromptC 采用**路径作为 Skill 的唯一类型标识**：

```yaml
# PromptC-IR 运行时格式（兼容 AgenticDSL 规范）
### PromptC-IR '/lib/web/search@v2'
type: skill_node
signature:
  inputs:
    - name: query
      type: string
      path: "/lib/types/query"  # 类型也是路径！
  outputs:
    - name: results
      type: array
  # 关键：description 用于动态路由匹配
  description: "Search the web for academic papers using keywords"
  
sandbox_level: L1  # 替代 effect system
permissions:
  - network: { domains: ["arxiv.org", "scholar.google.com"] }
  - llm_call: { model: "gpt-4", budget_tokens: 2000 }

implementation:
  type: compiled  # compiled | interpreted | llm_template
  binary_ref: "sha256:abc123..."  # 固化后的 WASM/Native 代码
  # 或 LLM 模板（未固化时）
  prompt_template: "Search query: {{ $.query }}"
```

**路径层级语义**（融合 AgenticDSL 安全模型）：

| 路径前缀 | 可写性 | 沙箱默认 | 编译策略 | 用途 |
|----------|--------|----------|----------|------|
| `/lib/**` | ❌ 只读 | **L0**（信任） | AOT 编译至机器码 | 已固化的标准 Skill，类似 AgenticDSL 的标准库 |
| `/app/**` | ✅ 允许 | **L0/L1**（开发时 L0，生产可提升） | JIT 编译 + 缓存 | 应用特定 Skill，工程代码 |
| `/dynamic/**` | ✅ 自动 | **L1/L2**（运行时生成默认 L1） | 解释执行 / LLM 生成 | 运行时 LLM 自优化生成的子图 |
| `/cache/**` | ✅ 自动 | **L0**（纯数据） | 无 | 执行缓存、上下文快照 |

## 3. 自动化移植工具：Skill Bridge（技能桥）

专门解决 OpenClaw/Claude/MCP → PromptC 的迁移问题，**保留 description 用于动态路由**：

### 3.1 移植流程

```python
# 伪代码：CLI 工具 promptc-migrate
class SkillBridge:
    def import_openclaw(self, skill_md_path: str) -> PromptC_IR:
        """
        输入：OpenClaw SKILL.md
        输出：PromptC-IR JSON（位于 /dynamic/migrated/ 下）
        """
        # 1. 解析 Markdown 中的 YAML frontmatter
        meta = parse_frontmatter(skill_md_path)
        
        # 2. 提取关键字段映射
        ir_node = {
            "path": f"/dynamic/migrated/{meta['name']}@v1",
            "type": "skill_node",
            "signature": {
                "inputs": meta["args"],
                "outputs": [{"name": "result", "type": "any"}],
                # 保留 OpenClaw 的 description 用于动态匹配！
                "description": meta["description"],  
                "capabilities": meta.get("capabilities", [])
            },
            # 沙箱映射：OpenClaw 的 checkPermissions -> L1/L2
            "sandbox_level": self._map_permissions(meta["checkPermissions"]),
            "permissions": self._extract_permissions(meta),
            
            # 实现体：初始为 LLM 模板（解释执行）
            "implementation": {
                "type": "llm_template",
                "prompt": meta["prompt"],
                "engine": meta.get("model", "gpt-4")
            }
        }
        return ir_node

    def _map_permissions(self, perms: List[str]) -> str:
        """将 OpenClaw 权限映射到沙箱级别"""
        if "code_interpreter" in perms or "python3" in perms:
            return "L2"  # 代码执行必须 L2
        elif "web_search" in perms or "http" in perms:
            return "L1"  # 网络访问 L1
        return "L0"
```

### 3.2 渐进式固化（Compilation-as-a-Service）

迁移后的 Skill 初始位于 `/dynamic/`，通过**执行热度检测**自动升级至 `/lib/`：

```yaml
### PromptC-IR /__meta__/compilation
compilation_policy:
  hot_spot_detection:
    threshold_calls: 100  # 100 次调用后触发编译
    threshold_latency: 500ms  # 或平均延迟超过 500ms
    
  promotion_path:
    source: "/dynamic/migrated/web_search@v1"
    target: "/lib/web/search@v2"  # 提升为官方库路径
    
  compilation_backend:
    - type: "wasi"  # 先编译为 WASM（跨平台）
    - type: "native"  # 热点进一步编译为本机代码
    
  # 关键：固化后保留 description 签名，确保动态路由仍能发现
  signature_preservation: true
```

## 4. 用沙箱级别替代效应系统（Sandbox > Effect）

简化 PromptC 的效应系统，采用 AgenticDSL 的**三级沙箱 + 显式权限**模型：

### 4.1 沙箱语义映射

原 PromptC 的 Effect System → 新沙箱模型：

| 原 PromptC Effect | 新沙箱级别 | 权限声明 | 运行时行为 |
|-------------------|------------|----------|------------|
| `emits Pure` | **L0** | `[]` | 同进程执行，无隔离开销 |
| `emits Network` | **L1** | `[network: {domains: [...]}]` | Seccomp-BPF 限制，仅允许指定域名 |
| `emits IO` | **L1/L2** | `[fs: {read: [...], write: [...]}]` | L1 只读，L2 写操作需 OverlayFS |
| `emits LLMCall` | **L0**（模型推理视为安全） | `[llm: {model: ..., budget: ...}]` | 通过 AgentPool 申请资源，受预算控制 |
| `emits CodeExec` | **L2** | `[runtime: {name: python3, imports: [...]}]` | Namespace 隔离，禁止访问 AgentPool |

### 4.2 源码层语法（简化版）

```promptc
# /app/my_analysis.promptc
# 不再写复杂的 effect 签名，改用沙箱注解

@sandbox(L1)  # 显式声明，或自动推断
skill WebSearch<query: String> -> Results {
    # L1 允许网络，但默认只允许 HTTPS GET
    permissions: {
        network: { methods: ["GET"], domains: ["*.arxiv.org"] }
    }
    
    # 实现：可以是外部工具调用或 LLM
    implementation: external("mcp::web_search")
}

@sandbox(L2)  # 代码执行必须 L2
skill CodeInterpreter<code: String, data: CSV> -> Analysis {
    permissions: {
        runtime: { 
            name: "python3",
            allow_imports: ["pandas", "numpy"],
            timeout_sec: 30
        }
    }
    
    # L2 沙箱通过 IPC 与主进程通信，无法直接访问 LLM
    implementation: codelet("python3", sandbox_isolation: true)
}
```

## 5. 上下文管理：路径级 COW + 增量快照

完全借鉴 AgenticDSL 的**写时复制（COW）**与**快照机制**，解决 LLM 上下文长度爆炸问题：

### 5.1 执行上下文模型

```yaml
### PromptC-IR 执行实例示例
execution_instance:
  id: "exec_main_abc123"
  session_id: "session_user_xyz"
  
  # 上下文分层：只有本路径下的节点可访问
  context_layers:
    global: {}  # 全局常量
    "/main/input": { query: "量子计算" }
    "/main/fork/step1": { partial_result: [...] }  # Branch A
    "/main/fork/step2": { partial_result: [...] }  # Branch B
    
  # 增量快照：通过路径引用历史状态，而非复制全部
  snapshots:
    "/main/step3": { hash: "sha256:def...", ref: "ctx_snapshots['/main/step3']" }
```

### 5.2 在 PromptC 源码中使用上下文

```promptc
# /app/research_assistant.promptc
skill DeepResearch<topic: String> -> Report {
    # 1. 起始节点，写入 /main/input
    node search: WebSearch(topic) 
        -> "/main/search/results";
    
    # 2. Fork 并行分析（创建 COW 分支）
    fork "/main/analysis" {
        # Branch A: 技术角度
        branch tech: AnalyzeTech("/main/search/results[0]") 
            -> "/main/analysis/tech";
            
        # Branch B: 商业角度  
        branch biz: AnalyzeBusiness("/main/search/results[1]")
            -> "/main/analysis/biz";
    }
    
    # 3. Join 合并：通过路径显式选择合并策略
    join "/main/analysis" {
        strategy: "deep_merge",  # AgenticDSL 的合并策略
        fields: {
            "tech.insights": "array_concat",
            "biz.market_size": "last_write_wins"
        }
    }
    
    # 4. 访问历史快照（减少上下文传输）
    node compare: CompareWithPrevious(
        current: "/main/analysis",
        previous: "$.ctx_snapshots['/main/previous_run']"  # 引用上次执行
    );
    
    return compare.result;
}
```

**关键优化**：当向 LLM 请求生成子图时，**只传递相关路径的上下文**，而非全量历史：
```yaml
# LLM 生成请求时的上下文裁剪
prompt_context:
  include_paths: ["/main/search/results", "/main/analysis/tech"]
  exclude_paths: ["/main/analysis/biz"]  # 不相关分支
  max_tokens: 4000
```

## 6. Skill Description 匹配与动态路由

保留 OpenClaw/Claude 的**描述匹配**能力，实现真正的动态 Skill 发现：

### 6.1 运行时 Skill 注册表

```yaml
### PromptC-IR /__meta__/skill_registry
registry:
  # 静态注册（已编译）
  "/lib/web/search@v2":
    description: "Search academic papers from arXiv"
    capabilities: ["search", "academic"]
    sandbox_level: L1
    
  # 动态注册（运行时生成或迁移）
  "/dynamic/migrated/sales_analysis@v1":
    description: "Analyze sales data from CSV and generate report"
    capabilities: ["data_analysis", "csv"]
    sandbox_level: L2
    
  # LLM 可根据任务描述自动选择
  auto_route:
    enabled: true
    fallback: "/lib/general/llm_call@v1"
```

### 6.2 动态路由节点（类似 AgenticDSL 的 generate_subgraph）

```promptc
# 使用描述匹配动态选择 Skill
skill DynamicAgent<task: String, data: Any> -> Result {
    # 1. 分析任务，匹配最佳 Skill
    node selector: SkillMatcher(
        task_description: task,
        candidate_paths: ["/lib/**", "/app/specialized/**"],
        strategy: "description_similarity"  # 基于嵌入向量匹配
    ) -> "/main/selected_skill";
    
    # 2. 动态调用（路径即类型）
    node execute: DynamicCall(
        skill_path: "/main/selected_skill",  # 运行时确定的路径
        input: data,
        # 关键：增量上下文，只传递 data 路径
        context_scope: ["/input/data"]
    );
    
    return execute.result;
}
```

## 7. 完整示例：迁移后的研究助手

展示从 OpenClaw 迁移到 PromptC，并逐步固化至 `/lib` 的完整流程：

### 7.1 阶段一：自动迁移（/dynamic）

```yaml
### 由 SkillBridge 自动生成：/dynamic/migrated/academic_search@v1
type: skill_node
path: "/dynamic/migrated/academic_search@v1"
signature:
  description: "Search academic papers from arXiv using keywords"  # 保留 OpenClaw desc
  inputs:
    - name: keywords
      type: string
  outputs:
    - name: papers
      type: array
sandbox_level: L1
permissions:
  - network: { domains: ["arxiv.org"] }

implementation:
  type: llm_template  # 初始为解释型
  prompt: |
    Search arXiv for papers about {{ $.keywords }}.
    Return top 5 results with title and abstract.
```

### 7.2 阶段二：人工优化（/app）

```promptc
# /app/research_pipeline.promptc
# 人类开发者改进，添加类型安全和并行处理

@sandbox(L1)
skill AcademicSearch<keywords: String, top_k: Int = 5> -> Papers {
    # 显式权限更精细
    permissions: {
        network: { 
            endpoints: ["https://export.arxiv.org/api/query"],
            rate_limit: "10/min"
        }
    }
    
    # 混合实现：外部 API + LLM 摘要
    node raw: HttpRequest(
        url: "https://export.arxiv.org/api/query",
        params: { search_query: keywords, max_results: top_k }
    );
    
    node summarized: Map(
        collection: raw.entries,
        body: |entry| => LLMSummarize(entry.abstract),
        parallelism: 3,  # PromptC 特性：自动批处理
        sandbox_level: L0  # LLM 调用在 L0
    );
    
    return summarized.results;
}
```

### 7.3 阶段三：热点固化（/lib）

```yaml
### 编译器自动优化后：/lib/academic/search@v3
type: skill_node
path: "/lib/academic/search@v3"
signature:
  description: "Search academic papers from arXiv"  # 描述保持，确保路由兼容
  # 编译期特化：top_k 变为编译期常量，生成优化代码
  generics:
    - name: top_k
      type: int
      default: 5
      
sandbox_level: L0  # 固化后提升为 L0（已验证安全）
implementation:
  type: compiled_wasm
  binary: "base64:wasm_binary..."  # 编译后的 WASM
  optimizations:
    - "http_connection_pool"  # 连接池复用
    - "response_cache"  # 自动缓存
  
  # 保留 LLM 作为 fallback
  fallback_template: "Generate papers about {{ $.keywords }}"
```

## 8. 实施路线图

| 阶段 | 目标 | 交付物 |
|------|------|--------|
| **MVP** | 建立 IR 规范与迁移工具 | `promptc-ir` 规范 v1.0（JSON Schema）、`skill-bridge` CLI（支持 OpenClaw/MCP 导入） |
| **v1.0** | 沙箱运行时集成 | 基于 AgenticDSL 运行时的 PromptC-VM，支持 L0-L2 沙箱、COW 上下文 |
| **v1.5** | 渐进式编译 | JIT 编译器（WASM 后端）、热点检测、自动 `/dynamic` → `/lib` 提升 |
| **v2.0** | 生态闭环 | 标准库 `/lib/std`、可视化 IDE（路径图编辑）、LLM 自优化（基于 description 匹配生成子图） |

此方案**保留 PromptC 的编译期优化优势**（融合、批处理、渐进式固化），同时**获得 AgenticDSL 的工程成熟度**（沙箱安全、COW 上下文管理、路径命名空间），并通过 **Skill Bridge** 实现现有生态的零成本迁移。
