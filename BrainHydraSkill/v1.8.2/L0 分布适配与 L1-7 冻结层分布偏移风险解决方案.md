## 1. 问题描述 (Problem Description)

### 1.1 核心风险：分布偏移 (Distribution Shift)
在 Hydra-SKILL v1.8.2 架构中，**L0 输入适配层**在 Stage 2/3 期间可训练（用于适配 CFI 回流数据分布），而 **L1-7 感知层**在 Stage 1 后冻结（权重 + LayerNorm 统计量）。
*   **现象**：CFI 回流数据（Type 1/2/3）经 L0 适配后，输出分布发生变化。但 L1-7 的 LayerNorm 仍使用 Stage 1 的统计量（Mean/Var）及初始仿射参数（$\gamma, \beta$）进行标准化。
*   **后果**：导致 **Activation Shift**，L1-7 提取的特征质量下降，尤其在 **推理阶段的多轮对话/CFI 回流场景** 下，模型对工具返回结果的敏感度降低，产生幻觉或忽略关键信息。
*   **文档依据**：架构评审报告 Sec 1, v1.8.2 Sec 2.1。

### 1.2 训练与缓存的矛盾
*   **训练需求**：为消除分布偏移，需允许 L1-7 的 LayerNorm 仿射参数（$\gamma, \beta$）在 Stage 2/3 期间微调。
*   **缓存矛盾**：若 $\gamma, \beta$ 更新，基于旧参数计算的 **Prefix Cache (KV Cache)** 将失效。若每次更新都重算 L1-7，将丧失外部递归的性能优势（v1.8.2 Sec 2.2）。
*   **委员会决策**：采用 **"Cache 存储不含 affine 中间态 + affine 单独前向"** 方案，即在 Stage 2/3 训练期间，缓存 LayerNorm 标准化后但未应用仿射变换的中间态，以支持仿射参数更新而不失效缓存。

---

## 2. 建议方案 (Proposed Solution)

### 2.1 架构策略总览
采用 **阶段化冻结与缓存策略**，区分训练与推理、Stage 1 与 Stage 2/3 的行为。

| 阶段 | L0 适配层 | L1-7 权重 | L1-7 LayerNorm Affine | Prefix Cache 策略 | 目标 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Stage 1** | 冻结 | 训练 | 训练 | 不使用 (全量计算) | 建立基础分布与 Cache |
| **Stage 2/3** | **训练** (Warmup) | **冻结** | **微调** | **存储 Pre-Affine 中间态** | 适配 CFI 分布，保护 Cache |
| **推理** | 冻结 | 冻结 | 冻结 | 标准 KV Cache (全冻结) | 高性能低延迟 |

### 2.2 关键机制设计
1.  **L0 门控 Warmup**：前 1000 步强制 `gate=1.0`，防止初期梯度消失（v1.8.2 Sec 2.1）。
2.  **L1-7 仿射参数解冻**：仅解冻 LayerNorm 的 `weight` ($\gamma$) 和 `bias` ($\beta$)，保持权重矩阵 $W$ 冻结。
3.  **Pre-Affine Cache (Stage 2/3 专用)**：
    *   缓存内容：LayerNorm 标准化后的状态 $\hat{x} = \frac{x - \mu}{\sigma}$。
    *   前向流程：从 Cache 读取 $\hat{x}$ → 应用当前 $\gamma, \beta$ → 计算 QKV → Attention。
    *   优势：$\gamma, \beta$ 更新无需重算 L1-7 主体权重，仅需重算仿射变换部分。

## 3. 论证思路 (Argumentation Logic)

### 3.1 认定的事实 (Facts)
1.  **L1-7 冻结必要性**：v1.8.2 Sec 2.2 明确 L1-7 Prefix Cache 永驻是外部递归性能基石。若权重更新，Cache 必须重建，延迟增加 3-5 倍。
2.  **分布偏移客观存在**：自然语言 (Type 0) 与工具返回 (Type 1/2/3) 的 Embedding 统计特性不同（v1.8.1 Sec 2.1）。
3.  **LayerNorm 特性**：`nn.LayerNorm` 输出 $y = \hat{x} \cdot \gamma + \beta$。若 $\gamma, \beta$ 变化，$y$ 变化，进而导致 KV Cache 变化。
4.  **委员会决策**：已确认采用 "Cache 存储不含 affine 中间态" 方案以平衡适配性与缓存有效性。

### 3.2 依赖的条件 (Conditions)
1.  **推理模式**：生产环境必须使用 `eval()` 模式，此时 L1-7 全部冻结，使用标准 KV Cache，无额外开销。
2.  **训练阶段**：仅在 Stage 2/3 启用 Pre-Affine Cache 机制。Stage 1 不使用 Cache 以保证梯度完整性。
3.  **CFI 频率**：若任务密集调用 CFI，L0 适配层必要性增加，L1-7 仿射微调收益显著。

### 3.3 问题场景保护 (Scenario Protection)
| 场景 | 风险 | 解决方案保护机制 |
| :--- | :--- | :--- |
| **推理 - 首轮对话** | 低 | L0 门控接近 0，输入分布与 Stage 1 一致，标准 Cache 有效。 |
| **推理 - 多轮/CFI 回流** | 高 (分布偏移) | L0 适配层激活。因推理时 L1-7 仿射已冻结，分布已适配完成，标准 Cache 有效。 |
| **训练 - Stage 2/3** | 高 (Cache 失效) | **Pre-Affine Cache 机制**：缓存 $\hat{x}$，动态应用 $\gamma, \beta$，支持仿射参数更新而不失效缓存。 |
| **训练 - 初期 (Step<1000)** | 高 (梯度消失) | **L0 Gate Warmup**：强制 `gate=1.0`，确保适配层梯度流通（v1.8.2 Sec 2.1）。 |

### 4.2 L1-7 LayerNorm 改造 (支持 Pre-Affine 输出)
```python
class AdaptableLayerNorm(nn.Module):
    """
    v1.8.2-Fix-01: 支持返回 Pre-Affine 中间态，用于 Stage 2/3 缓存
    """
    def __init__(self, hidden_size, eps=1e-6):
        super().__init__()
        self.weight = nn.Parameter(torch.ones(hidden_size))
        self.bias = nn.Parameter(torch.zeros(hidden_size))
        self.eps = eps
        
    def forward(self, hidden_states, return_pre_affine=False):
        # 1. 标准化 (统计量在 eval 模式下使用 running_mean/var)
        variance = hidden_states.pow(2).mean(-1, keepdim=True)
        hidden_states = hidden_states * torch.rsqrt(variance + self.eps)
        
        if return_pre_affine:
            # Stage 2/3 训练：返回标准化后但未应用 affine 的状态
            # 缓存此状态，affine 在外层应用
            return hidden_states 
        else:
            # 标准模式 / 推理：应用 affine
            return self.weight * hidden_states + self.bias
```

### 4.3 双缓冲区管理器修正 (支持 Pre-Affine Cache)
```python
class HydraMemoryManager:
    def __init__(self, stage="inference"):
        self.stage = stage  # 'stage1', 'stage2_3', 'inference'
        # Buffer 存储结构根据 stage 动态调整
        if stage == "stage2_3":
            # 存储 Pre-Affine 状态 [session, 7, 2048, 1152]
            self.prefix_buffer = torch.cuda.FloatTensor(max_sessions, 7, 2048, 1152)
        else:
            # 存储标准 KV Cache (MLA 压缩)
            self.prefix_buffer = torch.cuda.FloatTensor(max_sessions, 7, 2048, 256)
            
    def get_prefix_state(self, session_id, layer_idx):
        state = self.prefix_buffer[session_id, layer_idx]
        
        if self.stage == "stage2_3":
            # 从缓存读取 Pre-Affine 状态，动态应用当前 affine 参数
            # affine_params 需从模型层实时传入
            return state 
        else:
            # 推理/Stage 1: 直接返回 KV
            return state

    def save_prefix_state(self, session_id, layer_idx, state, affine_params=None):
        if self.stage == "stage2_3":
            # 缓存 Pre-Affine 状态
            self.prefix_buffer[session_id, layer_idx] = state.detach()
        else:
            # 缓存标准 KV
            self.prefix_buffer[session_id, layer_idx] = state.detach()
```

### 4.4 L1-7 层前向逻辑修正
```python
class L1_7_Layer(nn.Module):
    def __init__(self, config, stage="inference"):
        self.norm = AdaptableLayerNorm(config.hidden_size)
        self.stage = stage
        
    def forward(self, hidden_states, cache_manager, session_id, layer_idx):
        if self.stage == "stage2_3" and cache_manager.has_cache(session_id):
            # 训练 Stage 2/3: 读取 Pre-Affine 状态
            normed = cache_manager.get_prefix_state(session_id, layer_idx)
            # 动态应用当前 affine 参数 (支持梯度更新)
            hidden = self.norm.weight * normed + self.norm.bias
        else:
            # 推理/Stage 1: 标准流程
            hidden = self.norm(hidden_states)
            # 缓存逻辑 (略)
            
        # 继续 Attention/FFN 计算 (权重冻结)
        output = self.attention(hidden) + self.ffn(hidden)
        return output
```
