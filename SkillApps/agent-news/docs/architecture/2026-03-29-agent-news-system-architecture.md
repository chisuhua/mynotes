## 1. 系统概述

### 1.1 愿景与目标

**愿景**: 构建一个基于 OpenClaw 的自动化领域讯息抓取系统，实现智能体研究领域的全渠道、每日更新、智能去重、多级输出。

**核心目标**:
1. **全渠道抓取** — 论文/代码/博客/社交媒体
2. **每日自动更新** — 每日 9:00 自动执行
3. **智能去重** — 指纹去重 + 标题/URL 去重
4. **多级输出** — 原始链接 + 摘要 + 结构化存储 + 分析报告
5. **领域可扩展** — 配置文件支持多领域切换

**成功指标**:
- 覆盖 4+ 数据源
- 去重准确率 >95%
- 每日 9:00 前生成报告
- 支持 4+ 领域配置

## 3. 模块设计

### 3.1 爬虫层模块

| 模块 | 职责 | 数据源 |
|------|------|--------|
| `arxiv_crawler` | 爬取 arXiv 论文 | arXiv API |
| `github_crawler` | 爬取 GitHub Trending | GitHub Web |
| `blog_crawler` | 爬取技术博客 | RSS Feeds |
| `social_crawler` | 爬取社交媒体 | Twitter/Reddit API |

**设计原则**:
- 每个爬虫独立实现，统一接口
- 支持速率限制
- 支持错误重试

---

### 3.2 处理层模块

| 模块 | 职责 | 算法 |
|------|------|------|
| `deduplicator` | 去重处理 | 指纹去重 + 标题/URL 去重 |
| `summarizer` | 摘要生成 | OpenClaw 内置模型 |
| `classifier` | 自动分类 | 关键词匹配 + 规则分类 |

**去重策略**:
```python
# 指纹去重
fingerprint = hash(title + url + date)

# 标题去重
if title in seen_titles:
    skip()

# URL 去重
if url in seen_urls:
    skip()
```

---

### 3.3 存储层模块

| 存储类型 | 用途 | 格式 |
|---------|------|------|
| SQLite | 结构化存储 | 数据库表 |
| Markdown | 报告输出 | .md 文件 |
| JSON | 原始数据缓存 | .json 文件 |

**数据库 Schema**:
```sql
CREATE TABLE news_items (
    id INTEGER PRIMARY KEY,
    title TEXT,
    url TEXT UNIQUE,
    source TEXT,
    domain TEXT,
    summary TEXT,
    category TEXT,
    fingerprint TEXT,
    created_at DATETIME
);
```

## 4. 数据流

### 4.1 完整数据流

```
1. 调度器触发 (每日 9:00)
    ↓
2. 读取领域配置 (domains-config.yaml)
    ↓
3. 并行执行爬虫 (arXiv/GitHub/Blogs/Social)
    ↓
4. 去重处理 (Deduplicator)
    ↓
5. 摘要生成 (Summarizer)
    ↓
6. 自动分类 (Classifier)
    ↓
7. 存储数据 (SQLite + Markdown)
    ↓
8. 生成报告 (原始链接 + 简报 + 分析)
    ↓
9. 输出到 .acf/status/daily/
```

## 5. 错误处理策略

### 5.1 错误类型

| 错误类型 | 处理方式 |
|---------|---------|
| 爬虫失败 | 重试 3 次，记录错误日志 |
| API 限流 | 等待 + 降级（使用缓存） |
| 去重失败 | 跳过该项，继续处理 |
| 存储失败 | 回滚事务，发送告警 |

### 5.2 降级策略

```
API 不可用 → 使用缓存数据
摘要生成失败 → 使用原文前 200 字
分类失败 → 标记为"未分类"
```
